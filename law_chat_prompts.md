# law_chat.py 개발 프롬프트 전체 흐름

## 사전 준비 (pdf_retriver_st.py → pdf_retriver_st_persist.py)

1. pdf_reader.py 기반으로 st 붙여서 만들어라
2. Chroma 파일로 저장, 이미 인제스천된 파일은 재사용
3. 이미 있으면 "이미 참조데이터로 가지고 있습니다" 표시

## company_chat.py (내규 챗봇 — law_chat 전 단계)

4. 내규 문서에 대해 질문 답하는 챗봇
5. 고정 PDF 폴더 + 추가 업로드
6. 채팅 UI + 대화 맥락 유지
7. LCEL로 전환
8. 이미 인제스천 시 메시지 표시

## law_chat.py (본격 법률 자문)

9. 건축 법률 자문 도우미로 전환
10. 국내(건축법 해석례 2023/2024) + 해외(일본 BSL) 문서 인제스천
11. 채팅 히스토리 누적 — 세션마다 독립
12. 문서에 없으면 웹 검색 후 근거/출처 제시
13. 최근 변동사항도 항상 검색
14. 자주 묻는 질문 카테고리 선택 목록
15. 사이드바: 벡터 청킹 현황 + 자주 묻는 질문 나열
16. 문서 현황 폴딩 (국내N 해외N)
17. 청크 설정 UI 제거
18. DuckDuckGo → 국가법령정보센터 + Japanese Law Translation 공식 사이트
19. 법률 문서 디렉토리 `law_docs/` 분리
20. unsu.pdf 벡터DB에서 제거
21. LCEL 체인으로 전환
22. 스트리밍 출력
